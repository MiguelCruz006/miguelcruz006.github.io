Contexte et objectifs du projet EvolInfra
Analysé

En tant qu'assistant du responsable Système et Réseaux d'IT Services 86, la mission consiste à étudier, maquetter et valider une solution d'agrandissement du site de Chasseneuil de TiersLieux86. Le technicien informatique de TiersLieux86 n'a ni le temps ni les compétences pour se pencher sur ces questions d'infrastructure réseau.

Objectifs — Phase 1 : Gestion et sécurité
Homogénéiser la configuration de base de tous les équipements actifs (routeur + commutateurs).
Mettre en place un serveur NTP pour la synchronisation horaire des équipements.
Mettre en place un serveur Syslog pour la centralisation des journaux réseau.
Déployer un système VLAN pour cloisonner les réseaux des junior entreprises (ValorElec, Esporting) et espaces mutualisés.
Configurer le protocole VTP pour propager les configurations VLAN facilement entre commutateurs.
Mettre en place des procédures TFTP pour la sauvegarde/restauration des configurations et IOS.
Renforcer la sécurité : SSH v2, bannière légale, chiffrement des mots de passe.
Objectifs — Phase 2 : Tolérance de pannes
Recenser les SPOF (Single Point of Failure) de l'infrastructure et proposer des solutions.
Améliorer la bande passante inter-commutateurs avec EtherChannel (agrégation de liens).
Mettre en place RSTP (Rapid Spanning Tree) pour la tolérance de pannes des commutateurs.
Mettre en place HSRP pour la tolérance de pannes du routeur (passerelle redondante).
Contraintes
Utiliser les équipements Cisco existants — ajout de matériel uniquement pour la tolérance de pannes.
Durée d'interruption de service minimale lors des évolutions.
Maquette Packet Tracer obligatoire pour valider le fonctionnement avant déploiement réel.
Pour l'examen : maquette simplifiée avec 3 commutateurs et 2 routeurs minimum.

05SPOF · Audit
Tolérance aux pannes
Recensement des SPOF et préconisations d'évolution
Complété

Audit complet de l'infrastructure du site de Chasseneuil pour identifier les Single Points of Failure (SPOF) physiques et logiciels. Proposition d'un schéma réseau redondant préparant les missions 6, 7 et 8.

Tâche 1 — Problèmes physiques identifiés
Routeur unique (RoutChass) — si le routeur tombe, toute la connectivité inter-VLAN et Internet est perdue. Solution : ajouter un second routeur + HSRP (Mission 8).
Switch core unique (SwitchIntra) — si SwitchIntra tombe, tout le réseau est coupé. Solution : liens redondants + RSTP (Mission 7).
Lien FAI unique — un seul lien vers Internet. Solution préconisée : double FAI (hors périmètre de cet atelier).
Serveur SrvIntraDHCP unique — si le serveur tombe, plus de DHCP ni de NTP. Solution : serveur de secours ou failover DHCP.
Tâche 2 — Problèmes logiciels/système identifiés
Absence de redondance L2 — pas de Spanning Tree optimisé, convergence lente (30-50s en STP classique). Solution : RSTP/Rapid-PVST+ (Mission 7).
Bande passante limitée inter-commutateurs — liens Fa0/23-24 sous-utilisés. Solution : EtherChannel PAgP (Mission 6).
Pas de redondance de passerelle — IP de passerelle fixe sur RoutChass. Solution : HSRP avec IP virtuelle partagée (Mission 8).
Tâche 3 — Préconisations résumées

07RSTP · Rapid-PVST+
Tolérance pannes commutateurs — RSTP
Rapid Spanning Tree — convergence rapide <5s
Complété

Remplacement du STP classique (802.1D — convergence 30-50s) par Rapid-PVST+ (convergence <5s). SwitchIntra est élu Root Bridge pour tous les VLANs. PortFast et BPDU Guard activés sur les ports d'accès pour éliminer le délai de démarrage des postes clients.

Tâche 1 — Configuration Rapid-PVST+
SwitchIntra — Root Bridge + Rapid-PVST+
IOS
! ── Activation Rapid-PVST+ sur tous les switches ─────────────────────
SwitchIntra(config)# spanning-tree mode rapid-pvst
SwitchEtages(config)# spanning-tree mode rapid-pvst
SwitchDMZ(config)#   spanning-tree mode rapid-pvst

! ── SwitchIntra = Root Bridge pour tous les VLANs (priorité 4096) ───
SwitchIntra(config)# spanning-tree vlan 1-99 root primary
! → Équivalent à : spanning-tree vlan 1-99 priority 4096

! ── PortFast sur ports d'accès (SwitchEtages) ────────────────────────
! → Élimine le délai STP (listening/learning) pour les PCs
SwitchEtages(config)# spanning-tree portfast default
! → Active PortFast sur tous les ports non-trunk

! ── BPDU Guard — sécurité sur ports PortFast ─────────────────────────
! → Désactive le port si un BPDU est reçu (ex: switch non autorisé)
SwitchEtages(config)# spanning-tree portfast bpduguard default

08HSRP · L3
Tolérance pannes routeur — HSRP
Hot Standby Router Protocol — passerelle virtuelle redondante
Complété

Mise en place de HSRP entre RoutChass (actif, priorité 110) et RouterFAI (standby, priorité 100). En cas de panne du routeur actif, le basculement est automatique et transparent pour les utilisateurs — ils continuent de pointer vers la même IP virtuelle (192.168.2.254).

Tâche 1 — Configuration HSRP
RoutChass (Actif) + RouterFAI (Standby)
IOS
! ── RoutChass = routeur HSRP ACTIF (priorité 110) ────────────────────
RoutChass(config)# interface Fa0/0
RoutChass(config-if)# ip address 192.168.2.1 255.255.255.0
RoutChass(config-if)# standby 1 ip 192.168.2.254
! → IP virtuelle partagée — configurée comme passerelle sur tous les PCs
RoutChass(config-if)# standby 1 priority 110
! → Priorité 110 > 100 → RoutChass est élu ACTIVE
RoutChass(config-if)# standby 1 preempt
! → Preempt : RoutChass reprend le rôle ACTIVE automatiquement après restauration
RoutChass(config-if)# standby 1 timers 3 10
! → Hello toutes les 3s, dead timer 10s
RoutChass(config-if)# no shutdown

! ── RouterFAI = routeur HSRP STANDBY (priorité 100 par défaut) ───────
RouterFAI(config)# interface Fa0/0
RouterFAI(config-if)# ip address 192.168.2.2 255.255.255.0
RouterFAI(config-if)# standby 1 ip 192.168.2.254
! → Même IP virtuelle que RoutChass
RouterFAI(config-if)# standby 1 timers 3 10
RouterFAI(config-if)# no shutdown

! ── Vérifications ───────────────────────────────────────────────────
RoutChass# show standby brief
! → Interface  Grp  Pri  P  State   Active    Standby   Virtual IP
! → Fa0/0      1    110  P  Active  local     192.168.2.2  192.168.2.254
RouterFAI# show standby brief
! → Fa0/0      1    100     Standby 192.168.2.1  local  192.168.2.254
